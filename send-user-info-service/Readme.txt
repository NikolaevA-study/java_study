Задание:
Разработать 2 микросервиса, занимающихся сохранением информации о пользователе:

1) Первый микросервис получает информацию по REST и складывает ее в RabbitMQ или Kafka (можно выбрать один вариант, а можно реализовать оба и переключать между ними). Микросервис реализует создание, обновление и удаление записей (без получения).
2) Второй микросервис получает через RabbitMQ сообщение и производит необходимые действия в БД (PostgreSQL). Микросервис умеет по REST отдавать информацию о пользователях (всех списком и поиск по имени или фамилии).

Пример входящего сообщения для первого микросервиса:

{
    "id": 1,
    "name": "Alexandr",
    "surname": "Nikolaev",
    "age": 28,
    "contacts": [
        {
            "type": "PHONE",
            "value": "+79999999999"
        },
        {
            "type": "EMAIL",
            "value": "nikolaev@testapp.ru"
        }
    ]
}

Использовать: Spring Boot, Spring Data JPA, Spring AMQP, Mapstruct (если потребуется маппинг), Gradle, DB changelog Liquibase (в нотации yaml), Docker
